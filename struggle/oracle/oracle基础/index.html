安装步骤

  1. 在oracle 11g的几个版本中，企业版安装的选项比较全面，所以我们选择企业版安装，官方下载地址，选择合适自己电脑版本的安装包呀
  2. 下载完成，解压两个压缩包到同一目录（默认：database）
  3. 双击解压目录下的setuo.exe开始安装，静静地等待几秒钟后会弹出一个环境不满足最低要求，是否确实要继续？，点击是，直接安装就可以。
  4. 然后到达配置安全更新页面，这会让你输入邮箱用于接受有关安装更新问题的通知，**注意了！：**去掉那个勾（我希望巴拉巴拉。。。）邮箱也不用填，咱又没有Oracle的metlink账户，怎么可能连接成功呢！Oracle的metlink账户是要付费使用的，而且米米还不是一点点的！所以只要不填写直接走下一步就OK了。
  5. 然后到达选择安装选项，选择“创建和配置数据库”选项，然后点击下一步
  6. 然后是系统类”界面下选择“桌面类”，然后单击“下一步”安装。
  7. 进入“典型安装配置”界面，根据个人需要更改各项路径（注意路径中不要有中文，后面会报错）；随后输入管理口令并确认，需要注意的事，输入口令要求是至少包含大小写字母和数字的复杂密码形式,（**注意啦！**不要用数字或者符号开头，用大小写字母都行，不然后面会报错，账户会被锁定）
  8. 在“概要”界面中，显示安装信息的概要情况，然后点击“完成”按钮，等待程序的安装，安装过程可能会遇到ORA-00922：选项缺失或无效问题解决办法,这就是在设置实例密码时以数字开头了，如果忽略该问题继续的话，就会遇到ORA-28000: 账户锁定,如果继续忽略的话，安装完成的时候会有更改口令的选项，进去以后也可以解锁账号，之后再检查账号状态时候就是open了，应该不会有问题了
  9. 过程中可能还会遇到启动服务出现错误，找不到OracleMTSRecoveryService，解决办法：cmd-输入regedit运行注册表，然后到HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services，找到OracleMTSRecoveryService及OracleRemExecService，查看ImagePath对应路径是否正确,如果发现dbhome_2直接改为dbhome_1，可能是之前没卸载干净

客户端工具

SQL Plus工具：是 Oracle 系统默认安装的自带的一个客户端工具。在 Windows cmd 命令行中输入“sqlplus”命令，启动该工具，输入system和密码即可登录到本地数据库

可以通过命令行进行数据库链接：conn 用户名/密码@服务器连接字符串 as 连接身份,例如：conn sys/****@127.0.0.1:1521/orcl as sysdba，配置本地网络服务名后可直接通过网络服务名连接数据库，conn sys/密码@网络服务名 as 身份名，例如：conn sys/666666@dengqi as sysdba

oracle的三种身份：

  1. sysdba:数据库管理员身份。权限：打开（关闭）数据库服务器、备份（恢复）数据库、日志功能、会话限制、数据库管理功能等。例如：sys用户必须用sysdba才能登陆，system用户用普通用户就可以登陆。
  2. sysoper:数据库操作员身份 。权限：打开（关闭）数据库服务器、备份（恢复）数据库、日志功能、会话限制。
  3. normal:普通用户。权限：操作该用户下的数据对象和数据查询，默认的身份是normal用户。

为了使用规范，可以先修改一下设置错误的密码：alter user 数据库连接用户名 identified by 新密码;（注：1.分号不能忘；2.密码若为数字和字母组合，且数字开头，密码要用双引号）

在实际Oracle的开发过程当中，经常会使用到一个第三方开发工具PL/SQL Developer，因为SQLPLUS开发过程当中操作相对比较麻烦，不具有友好的可视化操作，因此，使用“PL/SQL Developer”开发工具替代SQLPLUS来进行oracle数据库开发，他的运行也需要安装Oracle客户端软件，并且可以通过本地网络服务配置向导配置好网络服务名，可以使用服务名直接连接登录

解决办法：该错误表示实例没有启动

  2. connect / as sysdba，连接到空闲例程
  3. startup
